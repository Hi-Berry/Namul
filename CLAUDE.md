# 조선 나물전기 — 작업 가이드 (Claude용)

웹 기반 픽셀 게임. 정적 사이트(`index.html` + `js/*` + `css/style.css` + `assets/`).
콘텐츠 대부분은 `js/data.js`의 데이터 테이블로 분리되어 있습니다.

## 폴더 개요
- `index.html` — 게임 진입점. JS 수정 후 맨 아래 `?v=숫자`를 +1 해야 캐시 갱신.
- `js/` — 게임 로직 (`data.js` 콘텐츠, `world.js`·`combat.js`·`trading.js` 등 시스템).
- `docs/` — **기획 문서.** `design-spec-snapshot.md`(기획서 기준 사양), `CONTENT.md`(콘텐츠 편집 가이드).
- `제작 히스토리.html` — 작업 내역을 날짜별로 일반인도 이해하기 쉽게 기록한 페이지.

## ⭐ "[기획]" 커밋 처리 워크플로 (자동 작업 규약)

이 저장소에 **제목이 `[기획]`로 시작하는 커밋**이 올라오면 다음 순서로 처리합니다.

1. **읽기** — `docs/` 폴더(특히 `design-spec-snapshot.md`, `CONTENT.md`)에서
   새로 "구현할 내용 / 수정할 내용"을 파악합니다. 커밋 메시지·변경된 문서의 diff를 함께 확인합니다.
2. **구현** — 해당 사양을 실제 코드(`js/`, `css/`, `assets/` 등)에 반영합니다.
   - JS를 고쳤다면 `index.html`의 `?v=` 버전을 올립니다.
   - 기획서 사양이 바뀌었다면 `docs/design-spec-snapshot.md`의 "변경 이력"·"정합 현황"도 갱신합니다.
3. **히스토리 기록** — `제작 히스토리.html`을 갱신합니다.
   - 날짜별로 묶되 **가장 최근 날짜가 맨 위**에 오도록 정렬합니다.
   - 새 기록은 파일 안의 `<!-- ▼▼▼ 새 작업 기록은 ... ▼▼▼ -->` 마커 **바로 아래**에 추가합니다.
   - 같은 날 작업이면 기존 그날 `section.entry` 안에 카드(`.card`)를 추가합니다.
   - 설명은 **게임을 모르는 일반인도 이해할 수 있도록** 쉬운 말로 풀어 쓰고,
     각 항목마다 "쉽게 말하면 —" 요약(`.plain`)을 답니다.
4. **푸시** — 작업 브랜치에 커밋 후 `git push -u origin <branch>`.
   (별도 요청이 없으면 PR은 만들지 않습니다.)
